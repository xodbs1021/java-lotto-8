# java-lotto-precourse

**🏆 로또 당첨 시뮬레이터 (Lotto Win Simulator)**

  💡 프로젝트 개요
  
이 프로젝트는 주어진 금액으로 로또 복권을 구매하고, 당첨 번호 및 보너스 번호를 입력받아 구매한 로또의 당첨 여부와 결과를 계산하는 콘솔 기반 시뮬레이터입니다.

핵심 비즈니스 로직은 클린 코드(Clean Code) 원칙과 미션 요구 사항을 엄격하게 준수하여 구현되었으며, 모든 기능은 JUnit 5 기반의 단위 테스트를 통해 검증되었습니다.


**🛠️ 개발 환경 및 실행 방법**

| 항목 | 상세 내용 |
| :--- | :--- |
| **개발 언어** | Java 21 |
| **빌드 도구** | Gradle |
| **테스트 도구** | JUnit 5, AssertJ |

**🚀 기능 목록 (정상 동작 및 예외 상황)**

| 기능 | 정상 동작 | 예외 상황 (Validation) |
| :--- | :--- | :--- |
| **구매 금액 입력** | 1,000원 이상의 금액을 입력하며, 1,000원 단위로 나뉘어 로또를 구매한다. | 1. **1,000원 미만**의 금액을 입력한 경우. <br> 2. **1,000원 단위**로 나누어 떨어지지 않는 경우. <br> 3. **숫자 외의 값**을 입력한 경우. |
| **로또 생성 및 출력** | 구매 개수만큼 1~45 사이의 중복되지 않는 6개 숫자를 생성하고, 번호를 오름차순으로 정렬하여 출력한다. | |


| 기능 | 정상 동작 | 예외 상황 (Validation) |
| :--- | :--- | :--- |
| **당첨 번호 입력** | 쉼표(`,`)로 구분된 6개의 숫자를 입력한다. | 1. 로또 번호가 **6개가 아닌** 경우. <br> 2. **중복된 번호**를 입력한 경우. <br> 3. 번호가 **1~45 범위**를 벗어난 경우. <br> 4. **숫자 외의 값**이 포함된 경우. |
| **보너스 번호 입력** | 당첨 번호에 포함되지 않은 1개의 숫자를 입력한다. | 1. **1~45 범위**를 벗어난 경우. <br> 2. **이미 당첨 번호에 포함된** 숫자인 경우. <br> 3. **숫자 외의 값**을 입력한 경우. |



| 기능 | 상세 동작 |
| :--- | :--- |
| **당첨 결과 계산** | 구매한 모든 로또를 당첨 번호 및 보너스 번호와 비교하여 등수를 판별하고 횟수를 누적한다. |
| **당첨 통계 출력** | 3개 일치(5,000원)부터 6개 일치(2,000,000,000원)까지 등수별 당첨 개수와 상금을 출력한다. (상금은 쉼표 포맷 적용) |
| **총 수익률 출력** | `(총 당첨 금액 / 총 구매 금액) * 100`을 계산하여 **소수점 첫째 자리**까지 반올림한 수익률을 출력한다. (예: `62.5%`) |


**📐 기술 요구 사항 준수**

| 요구 사항 | 준수 상세 내용 |
| :--- | :--- |
| **Indent Depth** | 모든 코드의 들여쓰기 깊이를 **2** 이하로 제한하여 코드의 복잡도를 낮추었습니다. |
| **함수 길이 제한** | 모든 함수(메서드)의 길이를 **15라인** 이하로 구현하여 한 가지 일만 하도록 책임을 분리했습니다. |
| **`else` 예약어 금지** | `else` 예약어를 사용하지 않고, `if` 조건절 내에서 `return` 또는 Guard Clause 패턴을 활용하여 로직을 구현했습니다. |
| **하드 코딩 방지** | 문자열 상수, 숫자 값, 로또 범위 등은 `staticWord` Enum 및 클래스 내 **상수**로 정의하여 명확한 의미를 부여했습니다. |
| **Java Enum 활용** | 당첨 등수, 상금 정보, 에러 메시지 관리 등 프로그램 내의 고정된 값 관리에 `Java Enum`을 적극적으로 적용했습니다. |
| **단위 테스트** | `camp.nextstep.edu.missionutils.test` 라이브러리를 활용하여 UI 로직을 제외한 모든 비즈니스 로직에 대한 단위 테스트를 작성했습니다. |
